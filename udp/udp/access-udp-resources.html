Accessing UDP Resources

Unizin Data Platform


Overview

Note: the Unizin Data Platform implementation is currently in an engineering "hardening" phase. We are migrating from a proof-of-concept implementation to a production-ready implementation of the UDP's ingestion capabilities. During this period, access to UDP resources is limited and focused on validation of the UDP's ingestion capabilities.

Access to UDP resources is presently authenticated through GSuite accounts managed by Unizin. You may request as many distinct GSuite accounts as you need to facilitate validation of data ingestion.

UDP Stores

Two key UDP resources can be accessed by Institutions at present:

  1. UDP Relational store. This relational data warehouse store captures object data from the teaching and learning ecosystem.. All data in the relational store is modeled in the Unizin Common Data Model. Furthermore, all data in the relational store unifies the representation of objects from across the landscape of teaching and learning tools.
  2. UDP Learning record store. The UDP Learning Record Store is a data lake of event data from across the teaching and learning tool ecosystem. Data in the Learning record store is modeled in the event record schema, which aligns with the Unizin Common Data Model. The UDP Learning Record Store is implemented in Google Big Query.

UDP Relational store

The UDP Relational warehouse store is implemented in Postgres inside Google Cloud SQL. The are various methods of connecting to Postgres in Google Clould SQL, most of which are supported by the UDP and your G Suite account.

Configuration  
Database name unizin The UCDM relational schema is applied to this database. Included in the relational schema are the UCDM data tables, the UCDM option set tables, and a tenant's UDP catalog tables.
Default IAM role cloudsql.viewer Provides read-only access to Cloud SQL using an IAM role native to the Google Clould Platform.

To conduct data validation, institutions will connect to their UDP relational store via a command line tool (e.g., psql) or using a standard JDBC/ODBC-compliant SQL client (e.g., SQL Workbench).

UDP Learning record store

The UDP's Learning record store is implemented in Google Big Query. There are a few different methods of interacting with Google Big Query. At present, Unizin supports interacting with Google Big Query via the Big Query command line tool, the REST API and client libraries.

Configuration  
Dataset name learning_datasets
Table name enriched_events
Default IAM role bigquery.user

Provides access with an IAM role native to the Google Clould Platform.

Provides permissions to run jobs, including queries, within the project. The user role can enumerate their own jobs, cancel their own jobs, and enumerate datasets within a project. Additionally, allows the creation of new datasets within the project; the creator is granted the bigquery.dataOwner role for these new datasets.
