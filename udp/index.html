What is the Unizin Data Platform?

The Unizin Data Platform (UDP) collects data from across the teaching and learning landscape and transforms it using the Unizin Common Data Model (UCDM). The UDP will eventually deliver services and products at the institutional and consortium level that enable a data-driven teaching and learning ecosystem. Expected UDP products and services include data marts, real-time event processing, and APIs in service of research, learning analytics, application development, and business intelligence.

The Unizin Common Data Model (UCDM) is the Unizin standard for representing data from across the teaching and learning ecosystem. Its purpose is to normalize data from a range of tools and systems that are critical to University researchers, faculty, application developers, and other staff who need a uniform vision of the data landscape. The UCDM aligns with and extends the Common Education Data Standards (CEDS). The UCDM also aligns with event standards such as Caliper and xAPI.

Note: the Unizin Data Platform is in active development. This includes the development of the UDP's ingestion capabilities. Access to data from the Unizin Data Platform is currently available as part of an "Early adopter" program. The function of this Early Adopter program is to work with Unizin members to articulate consumption patterns and use cases for data that can be built as consortium wide data products on the UDP.

Unizin members may request a UDP implementation.

The UCDM Data dictionary describes the UCDM Entities and Elements currently in use. Entity and element definitions provide a human-readable overview of the data inside an institutional UDP.

Relational schema

Complete documentation of the UCDM relational teaching & learning data schema. This documentation provides descriptions of every UCDM data table, table column definitions, and UCDM option sets if they are required for a particular column.

This document describes how Unizin maps Canvas Data to the UCDM Entities and elements. This documentation may be useful if you are used to pulling data from Canvas Data and wish to port to using the UCDM.
